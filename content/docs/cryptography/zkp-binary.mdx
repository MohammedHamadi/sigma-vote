# Disjunctive Binary Proofs

In a secure electronic voting system, we must ensure that every vote cast is valid (i.e., a "0" or a "1") without revealing the vote itself. If a voter could encrypt a "100", they could effectively hijack the entire election.

We use **Disjunctive Chaum-Pedersen Proofs** (also known as OR-proofs) to prove that a Paillier ciphertext $C$ contains either $0$ or $1$.

## The "OR" Problem

Standard Zero-Knowledge Proofs usually prove a single statement (e.g., "I know the secret $x$"). A Disjunctive Proof allows a prover to demonstrate that "Statement A is true **OR** Statement B is true," without revealing which one is actually true.

In our case:
- **Statement 0**: $C$ is an encryption of $0$ with randomness $r$.
- **Statement 1**: $C/g$ is an encryption of $0$ with randomness $r$.

## The Protocol (Fiat-Shamir)

To prove $m \in \{0, 1\}$ for $C = g^m r^N \pmod{N^2}$, the prover generates a proof $(\alpha_0, \beta_0, \gamma_0, \alpha_1, \beta_1, \gamma_1)$.

### 1. Proof Construction
The prover knows the real message $m$. Let's assume $m=1$:
1. **Simulation (for 0)**: Since $m \neq 0$, the prover "simulates" the proof for $0$ by picking random challenges and responses that satisfy the verification equation backwards.
2. **Real Proof (for 1)**: The prover generates a genuine proof for $1$ using their knowledge of the randomness $r$.

### 2. The Fiat-Shamir Heuristic
To make the proof non-interactive, the "challenge" is generated by hashing the commitments:
$$e = H(\text{modulus}, \text{ciphertext}, \text{commitment}_0, \text{commitment}_1)$$
The prover then splits this challenge $e = e_0 + e_1$, where one $e_i$ was used for the simulation and the other is derived for the real proof.

## Verification

The verifier (the SigmaVote server) checks:
1. **Combined Challenge**: $e_0 + e_1 \equiv H(...) \pmod N$
2. **Consistency (0)**: $v_0^N \equiv \alpha_0 \cdot C^{e_0} \pmod{N^2}$
3. **Consistency (1)**: $v_1^N \equiv \alpha_1 \cdot (C/g)^{e_1} \pmod{N^2}$

If all equations hold, the server is mathematically certain that $m$ is either $0$ or $1$, even though it remains encrypted.

## Why this is essential

Without this proof, a malicious actor could:
- **Vote Multiple Times**: By encrypting a value $>1$.
- **Subtract Votes**: By encrypting a negative value (or a very large value that behaves like a negative in modular arithmetic).

Binary proofs ensure the integrity of the tally while maintaining total voter privacy.
